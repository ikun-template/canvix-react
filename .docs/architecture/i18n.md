# 架构设计：国际化 (i18n)

## 概述

多语言资源统一维护在 `domains/locales`，按应用维度（editor / viewer）组织导出。运行时动态切换，非默认语言异步加载。

---

## 资源组织

```
domains/
└── locales/
    ├── package.json            # @ikun-kit/locales
    └── src/
        ├── langs/
        │   ├── zh-CN/
        │   │   ├── common.ts       # 通用文本（确认、取消、保存等）
        │   │   ├── canvas.ts       # 画布相关
        │   │   ├── sidebar.ts      # 侧边栏相关
        │   │   ├── inspector.ts    # 属性面板相关
        │   │   ├── toolbox.ts      # 工具箱相关
        │   │   └── widgets.ts      # 内置组件名称/描述
        │   ├── en-US/
        │   │   └── ...             # 同上结构
        │   └── ...                 # 其他语言
        ├── editor.ts               # editor 应用导出入口
        ├── viewer.ts               # viewer 应用导出入口
        └── index.ts                # 类型导出
```

### 导出入口

每个应用入口聚合该应用所需的模块：

```typescript
// editor.ts — 编辑器需要全部文本
export const editorMessages = {
  'zh-CN': () => import('./langs/zh-CN'),
  'en-US': () => import('./langs/en-US'),
};

// viewer.ts — 查看器只需 common + widgets
export const viewerMessages = {
  'zh-CN': () => import('./langs/zh-CN/common').then(/* 合并 widgets */),
  'en-US': () => import('./langs/en-US/common').then(/* 合并 widgets */),
};
```

- 导出为 `() => import()` 形式的异步加载函数
- 各应用只引入自己的入口，未使用的语言不参与打包

## 语言包格式

扁平 key-value 结构，key 使用 `模块.功能.描述` 的命名约定：

```typescript
// langs/zh-CN/sidebar.ts
export default {
  'sidebar.pages.title': '页面列表',
  'sidebar.pages.add': '新增页面',
  'sidebar.pages.delete': '删除页面',
  'sidebar.layers.title': '图层',
};
```

## I18n 能力提供

### I18nManager

```typescript
interface I18nManager {
  /** 当前语言 */
  getLocale(): string;
  /** 切换语言 */
  setLocale(locale: string): Promise<void>;
  /** 翻译 */
  t(key: string, params?: Record<string, string | number>): string;
  /** 获取支持的语言列表 */
  getSupportedLocales(): string[];
}
```

I18nManager 存放在 `EditorRefContext.config` 中，通过 `useI18n()` hook 消费。

### React Hook

布局模块和 widget 通过 Toolkit 使用：

```typescript
// Toolkit 导出
function useI18n(): {
  t: (key: string, params?: Record<string, string | number>) => string;
  locale: string;
};
```

```tsx
function SidebarHeader() {
  const { t } = useI18n();
  return <h2>{t('sidebar.pages.title')}</h2>;
}
```

### 参数插值

支持简单的参数替换：

```typescript
// 定义
'dialog.delete.confirm': '确定删除 {name} 吗？'

// 使用
t('dialog.delete.confirm', { name: '页面1' })
// → '确定删除 页面1 吗？'
```

## 语言切换

```
setLocale('en-US') 调用
  │
  ├─ 1. 异步加载目标语言包 (import)
  │
  ├─ 2. 替换当前语言资源
  │
  ├─ 3. 触发 hook: locale:changed
  │
  └─ 4. React re-render（通过状态更新触发所有 useI18n 消费者）
```

- 默认语言（如 `zh-CN`）可内联打包，无异步加载延迟
- 非默认语言首次切换时异步加载，加载后缓存

## 默认语言配置

默认语言在应用入口配置：

```typescript
// dock-editor main.ts
const i18n = createI18nManager({
  defaultLocale: 'zh-CN',
  messages: editorMessages,
});
```

## 包结构

```
domains/
└── locales/
    ├── package.json          # @ikun-kit/locales
    └── src/
        ├── langs/            # 各语言的翻译文件
        ├── editor.ts         # editor 导出入口
        ├── viewer.ts         # viewer 导出入口
        └── index.ts          # 类型导出

packages/
└── i18n/
    ├── package.json          # @ikun-kit/i18n
    └── src/
        ├── manager.ts        # I18nManager 实现
        ├── react.ts          # useI18n hook
        └── index.ts
```

- `@ikun-kit/locales`（domains）：语言资源，与业务相关
- `@ikun-kit/i18n`（packages）：i18n 运行时能力，与业务无关
