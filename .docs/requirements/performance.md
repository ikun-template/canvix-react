# 性能

## 核心问题

编辑器场景下同时存在大量组件实例，全量重新渲染不可接受。需在各层级实现最小运行时开销。

---

## 要求

### 组件级隔离

- 单个 Widget 的 Schema 变更不得触发同级其他 Widget 或父级的重新渲染

### 字段级精准更新

- 高频变更属性（位置、尺寸等空间属性）的更新应绕过框架的 diff 流程，直接作用于 DOM
- 非高频属性变更走常规渲染流程即可

### 挂载调度

- 大量 Widget 首次挂载时，需分批调度，避免长时间阻塞主线程
- 优先挂载视口内可见的 Widget

### 数据只读保护

- 对外暴露的 Schema 数据为只读，防止消费方意外修改导致状态不一致

### 按需加载

- 组件模块支持动态加载，未使用的组件不参与首屏加载
- 并发加载数量需可控，防止网络拥塞
