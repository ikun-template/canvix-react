# 数据模型

编辑器数据为三级实例层次结构：Document → Page → Widget。

---

## 三级实例

**Document（文档）**

- 全局唯一，代表一个完整的编辑文件
- 包含：schema 版本、标题、页面集合、全局资源（数据源等）

**Page（页面）**

- 隶属于 Document，一个文档包含多个页面
- 包含：唯一 ID、名称、画布尺寸、背景样式、组件集合

**Widget（组件）**

- 隶属于 Page 或另一个 Widget 的插槽
- 包含：唯一 ID、组件类型标识、空间属性、外观属性、组件自定义数据

---

## Widget 插槽

- Widget 可声明插槽（slot），每个插槽是一个可容纳子 Widget 列表的区域
- 一个 Widget 可拥有零个或多个具名插槽
- 插槽内的子 Widget 与 Page 下直属 Widget 具备相同的完整能力（含递归嵌套）
- 插槽机制使容器类组件（如布局容器、Tab 面板、卡片组等）成为可能

## 布局模式

- Widget 的渲染方式不固定为某一种布局方案
- 需支持至少两种布局模式：
  - **定位布局**：Widget 通过坐标绝对定位，自由摆放
  - **文档流布局**：Widget 按文档流排列
- 布局模式由 Widget 自身配置，决定"我自己"以何种方式渲染
- 仅需要切换渲染模式的 Widget 才需配置，其余使用缺省值

---

## 要求

- 每级实例拥有独立的、版本化的 Schema 定义
- Schema 是数据的唯一契约，驱动渲染、属性编辑、序列化等所有环节
- Schema 只承载渲染数据，编辑器运行时状态（缩放、滚动、选中等）不存入 Schema
- Schema 版本变更时需支持数据迁移
- 每个实例通过唯一 ID 标识，ID 在创建时生成、全文档内不重复
