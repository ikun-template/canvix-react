# Dock 底座与插件机制

## 定位

Dock 是编辑器/查看器的运行底座。Dock 自身不包含任何业务逻辑实现，仅提供：

1. 应用容器（入口初始化、布局挂载点）
2. 插件机制核心（注册、生命周期、依赖管理）
3. 运行时钩子（业务逻辑的执行切面）

所有业务功能（画布交互、属性编辑、图层管理等）由插件实现。

---

## 插件机制

### 注册

- 插件通过声明式注册，声明自身类型（layout / widget / service）和挂载目标
- 支持在应用启动前静态注册和运行时动态注册
- 插件注册信息包含：唯一标识、类型、依赖项、挂载配置

### 生命周期

- 插件拥有完整的生命周期：注册 → 初始化 → 挂载 → 激活 → 停用 → 卸载
- Dock 在各生命周期阶段调用对应的钩子函数
- 插件按注册顺序执行，销毁时逆序执行

---

## 运行时钩子

Dock 在以下时机提供钩子切面，插件可注册监听：

- **应用级**：初始化完成、销毁前
- **文档级**：文档加载完成、文档保存前/后、文档关闭前
- **变更级**：Operation 执行前（可拦截/修改）、Operation 执行后
- **视图级**：页面切换前/后、布局区域挂载/卸载

## 插件间通信

- 插件之间不直接引用，通过 Dock 提供的事件总线或 Toolkit 进行间接通信
- Dock 提供类型安全的事件发布/订阅能力
