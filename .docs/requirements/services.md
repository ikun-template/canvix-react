# 接口服务 (Services)

## 定位

Services 层为应用提供统一的数据接口调用，屏蔽底层存储差异。区分客户端（HTTP API）和服务端（本地存储）两套实现，通过门面层按运行时环境自动分发。

---

## 要求

### 接口抽象

- 所有数据操作通过 Services 层调用，上层不直接操作存储
- 数据模型（Models）为纯 TypeScript 类型定义，不含运行时逻辑
- 门面层对外暴露统一的接口，内部按环境分发到具体实现

### 环境分离

- **客户端实现（client）**：通过 HTTP API 调用远程服务，当前阶段仅预留接口骨架，不实现
- **服务端实现（server）**：本地存储，当前阶段基于 IndexedDB 实现，满足纯前端模板的演示需求

### 当前阶段范围

- 仅实现 server（IndexedDB）端，提供文档的增删改查能力
- client 端预留接口定义和骨架，后续对接真实后端时填充实现
- Services 层位于 `domains/services` 中
