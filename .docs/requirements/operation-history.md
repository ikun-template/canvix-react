# 变更与历史

## 变更模型

- 所有 Schema 数据的变更必须经过统一的变更入口，不允许直接修改数据对象
- 每次变更以 Operation 为单位，一个 Operation 描述一次原子变更或一组批量变更
- Operation 需包含足够信息以支持：应用变更、计算逆操作、广播通知

## 历史记录

- 每次变更自动计算并保存逆操作
- 支持 undo（撤销）和 redo（重做），操作粒度为单次变更调用
- 支持批量操作合并：连续的同类操作（如拖拽过程中的连续位移）可合并为一条历史记录
- 历史栈有容量上限，超出时丢弃最早的记录

## 变更广播

- 变更完成后需广播通知，通知内容包含本次变更的 Operation 详情
- 各消费方（布局模块、组件等）根据 Operation 内容自行判断是否需要响应
